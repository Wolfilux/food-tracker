# Security Policy

## Public Repository Rules

This repository is public. Treat every commit as world-readable.

- Never commit passwords, API keys, private keys, database dumps, production URLs with embedded credentials, or personal data.
- Store local configuration in \`.env\`; commit only \`.env.example\` with empty placeholders.
- Review generated files before committing.
- Prefer least-privilege tokens for integrations.
- Rotate and revoke any exposed secret immediately.

## Reporting

For security issues, contact the repository owner privately instead of opening a public issue with exploit details.
